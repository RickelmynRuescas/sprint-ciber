// 01 · Entrada e Validação

Toda entrada é hostil até provar o contrário.

Dados chegam do usuário, de outras APIs, de arquivos. Qualquer um pode ser armado. Validar, sanitizar e limitar antes de processar — sem exceção.

Todo input é uma superfície de ataque. Validar não é opcional — é a primeira linha.

SQL Injection Manipula queries para extrair ou destruir dados CRÍTICO
XSS Injeta scripts no browser de outros usuários CRÍTICO
Command Injection Executa comandos no servidor via input CRÍTICO
Payload Flooding Envia body gigante para travar parser ou consumir memória ALTO
Buffer Overflow Entrada além do buffer aloca memória fora do escopo ALTO

Tratamento Seguro de Erros

Erros não revelam stack trace, estrutura interna, banco, framework ou tecnologia usada.

Normalização de Parâmetros

Marca, modelo, versão e atributos seguem padrões definidos antes de chegar à regra de negócio.

exemplo · node/express 
// ✗ Jamais
db.query(`SELECT * FROM users WHERE id = ${req.params.id}`)

// ✓ Parameterized query + schema validation
const schema = Joi.object({ id: Joi.number().integer().positive() })
const { id } = schema.validate(req.params)
db.query('SELECT * FROM users WHERE id = ?', [id])

// ✓ Limitar tamanho de payload — 10kb máx
app.use(express.json({ limit: '10kb' }))

// 02 · Autenticação e Autorização

Saber quem é. Saber o que pode fazer.

JWT assina identidade. OAuth2 delega acesso. RBAC define o perímetro de cada perfil. Token sem expiração é credencial eterna — inaceitável.

Identidade verificada. Escopo definido. Cada perfil acessa exatamente o que precisa — nada além.

payload JWT seguro 
{
  "sub": "user_id_opaco",      // nunca expor PII
  "role": "analista",
  "iat": 1715000000,
  "exp": 1715000900,         // 15 minutos
  "jti": "uuid-único"          // revogação por ID
}
ciclo de renovação · token refresh 
// 1. Access token expira (15 min) — server responde 401
GET /api/leads  → 401 Unauthorized

// 2. Client envia refresh token (7 dias) para obter novo par
POST /auth/refresh  { "refresh_token": "rt_uuid_opaco" }

// 3. Server valida, invalida o refresh anterior (rotation)
//    e emite novo par — impede reutilização de token roubado
→ { "access_token": "...", "refresh_token": "novo_rt" }

MFA / TOTP

Segundo fator obrigatório para perfis críticos. TOTP (RFC 6238) gera código de 6 dígitos com janela de 30s. Senha vazada sozinha não garante acesso.

OAuth2 — Delegação Segura

Acesso a recursos externos sem expor credenciais. Authorization Code Flow com PKCE para SPAs. Escopos mínimos por cliente registrado.

// 03 · APIs e Serviços

A API é a superfície. Protege cada camada.

HTTPS garante confidencialidade. Rate limiting impede abuso. CORS define quem pode chamar. Payload hash verifica integridade.

HTTPS / TLS 1.3

Todo tráfego cifrado. HTTP redireciona para HTTPS. HSTS ativo. TLS 1.0 e 1.1 desabilitados. Sem cipher suites fracas.

Rate Limiting

100 req/min por IP em endpoints públicos. 10 req/min em /auth. 429 com Retry-After.

CORS Seguro

Whitelist explícita de origens. Credenciais apenas em domínios confiáveis. Sem *.

Integridade de Payload

HMAC-SHA256 no header X-Signature. Server rejeita payload sem assinatura válida.

A API é a fronteira entre usuário, serviços e banco. Cada requisição não autenticada é uma ameaça em potencial.

// ✓ Cliente — assinar payload antes de enviar
const sig = crypto.createHmac('sha256', SECRET_KEY)
  .update(JSON.stringify(body))
  .digest('hex');
req.headers['X-Signature'] = sig;

// ✓ Servidor — verificar antes de processar
const expected = crypto.createHmac('sha256', SECRET_KEY)
  .update(JSON.stringify(req.body))
  .digest('hex');

if (!crypto.timingSafeEqual(
  Buffer.from(req.headers['x-signature']),
  Buffer.from(expected)
)) return res.status(401).json({ error: 'invalid_signature' });
// 04 · Dados e Privacidade

Dado exposto é dado comprometido.

Criptografar em repouso e em trânsito. Manter só o necessário. Pseudonimizar o resto. Vazamento não é questão de "se" — é de "quando".

Criptografia em Repouso

AES-256 para dados sensíveis de clientes, leads e histórico de manutenção. bcrypt/Argon2 para senhas. Chaves em variáveis de ambiente, nunca no código.

Retenção e Descarte

Dados retidos só pelo tempo necessário. Soft delete com TTL. Purge automático via job agendado.

Pseudonimização

PII substituída por token opaco nos logs e relatórios — especialmente para modelos de ML e dashboards. Mapeamento original protegido separado.

Sem Exposição Acidental

Respostas de erro nunca retornam stack trace. Campos sensíveis excluídos do JSON de resposta por padrão.

// 05 · Logs e Auditoria

Sem log, não há investigação.

Log estruturado (JSON). Nível definido por evento. Sem dados sensíveis. Trilha de auditoria imutável. Alertas em tempo real para padrões suspeitos.

Logs existem para reconstruir incidentes. Sem trilha, não há forense — só suposição.

Estrutura JSON

Timestamp ISO 8601, nível, usuário anonimizado, ação, resultado. Parseable por SIEM. Cada campo é intencional — nenhum campo sensível.

Eventos Suspeitos

Brute force, acesso fora do horário, privilege escalation — alertas automáticos disparados por threshold configurável.

Trilha Imutável

Storage append-only — nenhum processo apaga registros. Retenção mínima de 90 dias para auditoria. Logs de incidentes: 1 ano.
